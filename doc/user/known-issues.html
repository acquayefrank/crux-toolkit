Known Issues

Version 1.38

July 19th, 2012
  1. Q-Ranker and Barista detect decoy PSMs by use of the --decoy-prefix string (ex. rand_) which is appended to the beginning of the peptide's protein ids. Currently, running the search-for-matches command with a fasta file generates protein ids from decoy PSMs without the "rand_" prefix. This issue will cause the following error for QRanker and Barista:
        FATAL: Cannot compute PEP without target or decoy scores.
    To prevent the error, run the search-for-matches with an index.
  2. The create-index command will fail when running on Cygwin. Crux will successfully create the index in a temporary directory but will report an error when it tries to rename the temporary directory: 
          WARNING: Cannot rename directory
      FATAL: Failed to create index
    The temporary directory will be named crux_XXXX where XXXX will consist of sequence of arbitrary letters and numbers. You can recover from the error by renaming the index directory manually.
  3. The create-index command may fail when running on Cygwin. Cygwin does not allow users to adjust the maximum allowed number of simultaneously open files. When the Crux reaches the open file limit, indexing will halt with the following message. 
          WARNING: cannot open all file handlers needed
      FATAL: Could not open file handle for index 746.
    You can still run search-for-matches directly on the fasta file, but this is significantly slower than a search using an index.
