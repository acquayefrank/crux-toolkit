Customization and Search Options

Crux allows the user the flexibility to change many of the search and analysis parameters. Attributes like the format of output and which peptides are selected from the protein database are controlled through the numerosu options. This page will start with some general information about options and then describe the use of several.

Introduction to options

A crux command is made up of three parts: executable name, required arguments, and options. Let's use a crux-search-for-matches command as an example. Here is the general form.

crux-search-for-matches [options] <ms2 filename> <protein filename>

In this case, the name is crux-search-for-matches. The required arguments are given inside of angle brackets, <>, and are the name of the input ms2 file and the name of the input protein fasta file. Options are just that, optional instructions. They are always placed between the executable name and the arguments. Any number of options can be included separated by spaces.

All of the available options are described for each executable on the documentation pages. You can also get a list of available options by run the command with no arguments.

crux-search-for-matches

You should see output that looks like this

Error in command line. Error # 5
the required argument <ms2> is missing.
Usage: match_search [options] <ms2> <fasta-file>
Optional Arguments:
  [--verbosity <int>] Specify the verbosity of the current processes from 0-100. (default = 10)
  [--match-output-folder <string>] The output folder to store the serialized binary output files. (default = .)
  [--output-mode <string>] The output format. sqt|binary|all (default = binary)
  [--sqt-output-file <string>] Name of the sqt file to place matches. (default = target.sqt)
  [--decoy-sqt-output-file <string>] Name of the sqt file to place matches. (default = decoy.sqt)
  [--spectrum-min-mass <double>] Spectra with m/z less than this value will not be searched. (default = 0)
  [--spectrum-max-mass <double>] Spectra with m/z greater than or equal to this value will not be searched. (default = 1e+09)
  [--parameter-file <string>] The crux parameter file to parse parameter from. (default = crux.params)
  [--score-type <string>] The type of scoring function to use. xcorr | xcorr_logp | sp_logp (default = xcorr)
  [--prelim-score-type <string>] The type of preliminary scoring function to use. sp (default = sp)
  [--number-decoy-set <int>] Specify the number of decoy sets to generate for match_analysis. (default = 2)
  [--spectrum-charge <string>] The spectrum charges to search. 1|2|3|all (default = all)
  [--number-runs <double>] The number of spectrum search runs to perform. (default = 1e+09)
Required Arguments:
  <ms2> The name of the file (in MS2 format) from which to parse the spectrum.
  <fasta-file> The name of the file (in fasta format) from which to retrieve peptides.

The first three lines are telling you that you forgot the required arguments and are reminding you what they are. The following lines list all the options. Crux options all begin with two dashes, --, followed by the name (with no space inbetween). Some options have their own arguments (all of the above do). The name is followed by a space and the appropriate argument. For example, to increase the verbosity to 20, we would include the option like this.

crux-search-for-matches --verbosity 20 sample.ms2 yeast.fasta
