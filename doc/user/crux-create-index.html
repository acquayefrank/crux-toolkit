crux-create-index

Description:

Given a protein fasta sequence database as input, generate an index of all of its peptides and save it to disk.

Usage:
crux-create-index [options]  <protein input file>  <index name>

Input:

  • <protein input file> – The name of the file (in fasta format) from which to parse proteins.
  • <index name> – The name of the directory where the newly created index will be placed.
Output:

    The main output of the program is an index for the fasta file  <protein input file> on disk. Hence, there is no STDOUT. The program prints to STDERR diagnostic messages informing the user of it's progress, depending on verbosity.
Options:
  • [--min-mass <float>] – The minimum neutral mass of the peptides to place in the index. Default=200
  • [--max-mass <float>] – The maximum neutral mass of the peptides to place in index. Default=7200
  • [--min-length <int>] – The minimum length of the peptides to place in the index. Default=6
  • [--max-length <int>] – The maximum length of the peptides to place in the index. Default=50
  • [--cleavages <tryptic | partial | all> ] – Type of cleavages to allow. The tryptic option outputs peptides that result from a complete in silico digestion of proteins after residues K or R except when followed by P, whereas all outputs all possible peptides. Default=tryptic
  • [--unique-peptides <T|F>] – Generate peptides only once, even if they appear in more than one protein if true (T). Default = F.
  • [--missed-cleavages <T|F>] – Allow missed cleavage sites with in a peptide. Default=T
  • [--isotopic-mass <average | mono>] – Specify the type of isotopic masses to use when calculating the peptide mass. Default=average
  • [--overwrite <T|F>] Replace existing files if true (T) or fail when trying to overwrite a file if false (F). Default=F.
  • [--parameter-file <string>] A file containing command-line or additional parameters. See the parameter documentation page for details.
  • [--verbosity <int>] – Specify the verbosity of the current processes from 0-100. Default=40.
Warning messages:
  • Fatal: The file "protein input filename" does not exist (or is not readable, or is empty).
Info:
  • All <float>s are supported up to 2 decimal digits
