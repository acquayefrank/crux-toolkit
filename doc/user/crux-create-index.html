create-index

Description:

Given a protein fasta sequence database as input, generate an index of all of its peptides on disk.

Usage:
create-index [options]  <protein input filename>

Input:

  • <protein input filename> – The name of the file (in fasta format) from which to parse proteins.
Output:

    The main output of the program is an index for the fasta file  <protein input filename> on disk. Hence, there is no STDOUT. The program prints to STDERR diagnostic messages informing the user of it's progress, depending on verbosity.
Options:
  • [--mass-range float] – The mass range contained in each index file. Default=1Da
  • [--min-mass float] – The minimum neutral mass of the peptides to place in the index. Default=200
  • [--max-mass float] – The maximum neutral mass of the peptides to place in index. Default=7200
  • [--min-length int] – The minimum length of the peptides to place in the index. Default=6
  • [--max-length int] – The maximum length of the peptides to place in the index. Default=50
  • [--cleavages tryptic|partial|all ] – Type of cleavages to allow. The tryptic option outputs peptides that result from a complete in silico digestion of proteins after residues K or R except when followed by P, wherease all outputs all possible peptides. Default=tryptic
  • [--redundancy redundant|unique] – Specify whether peptides that come from different proteins yet with identical sequences should appear on separate lines (redundant) or on the same line (unique). Default=unique
  • [--missed-cleavages] – Allow missed cleavage sites with in a peptide. Default=TRUE
  • [--isotopic-mass|average|mono] – Specify the type of isotopic masses to use when calculating the peptide mass. Default=average
  • [--verbosity int] – Specify the verbosity of the current processes from 0-100. Default=100
Warning messages:
  • Fatal: The file "protein input filename" does not exist (or is not readable, or is empty).
Info:
  • All floats are supported up to 2 decimal digits
  • User must ensure that file handler limit is sufficient enough for create-index. Use "ulimit -n [#file handlers]" on command line to increase number of allowed filehandlers. A typical default is 1024, should reset to ~20,000.
Bugs:
  • create-index does not have bugs, it has features.
