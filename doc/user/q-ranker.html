crux q-ranker

Usage:

crux q-ranker [options] <protein database> <search results directory>

Description:

Q-ranker dynamically learns to separate target from decoy PSMs. The algorithm is described in this article:

Marina Spivak, Jason Weston, Leon Bottou and William Stafford Noble. "Direct q value optimization methods for peptide identification from shotgun proteomics data sets." Journal of Proteome Research. 8(7):3737-3745, 2009.

The features used by q-ranker to represent each PSM are summarized here.

Input:

  • <protein input > – The name of the file in fasta format or the directory containing the protein index from which to retrieve proteins and peptides.
  • <search results directory> – A folder in which all the PSM result files are located. The program looks for files with names like '<fileroot>.<search>.target.txt' or '<fileroot>.<search>.decoy.txt', where '<search>' is either 'search' (produced by crux search-for-matches) or 'sequest' (produced by crux sequest-search). The search results should be from only one of the two search algorithms. All such files in the given directory are analyzed jointly.
Options:
  • -pi-zero <value> – The estimated proportion of target scores that are drawn according to the null distribution. Default=1.0.
  • --fileroot <string> – The fileroot string will be added as a prefix to all output file names. Default = none.
  • --output-dir <filename> – The name of the directory where output files will be created. Default = crux-output.
  • --overwrite <T|F> – Replace existing files if true (T) or fail when trying to overwrite a file if false (F). Default = F.
  • --parameter-file <filename> – A file containing command-line or additional parameters. See the parameter documentation page for details.
  • --no-xval – By default, q-ranker selects two training parameters, the learning rate and the weight decay, via cross-validation within the training set. This option turns off the cross-validation, leaving the parameters at their default values.
  • --feature-file <string> – Optional file in which to write the features. The features appear in the following order: xcorr, deltaCn, lnDeltaCn, Sp, logSp, deltaMass, abs(deltaMass), neutral mass, percent b/y ions matched, log(# candidates), UNUSED, UNUSED, missed cleavages, peptide length, charge = +1, charge = +2, charge = +3. Default = none.
  • --verbosity <0-100> – Specify the verbosity of the current processes. Each level prints the following messages, including all those at lower verbosity levels: 0-fatal errors, 10-non-fatal errors, 20-warnings, 30-information on the progress of execution, 40-more progress information, 50-debug info, 60-detailed debug info. Default = 30.
  • --version T – Print the version number and quit. Please note that you must include the 'T' after --version.
