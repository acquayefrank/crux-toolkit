crux-generate-peptides

Description:

Given a protein fasta sequence database as input, generate a list of peptides in the database that meet certain criteria (e.g. mass, length, trypticity) as output. The user can specify whether to use a pre-computed peptide index database.

Usage:
crux-generate-peptides [options]  <protein input filename>

Input:

  • <protein input filename> – The name of the file (in fasta format) from which to parse proteins.
Output:

    The program prints to standard output a series of header lines, describing how the peptides were generated, and according to what criteria, as follows: 
      # PROTEIN DATABASE: <protein input filename>
  # OPTIONS:  < options used to generate the file>
    Then the program prints a series of tab delimited lines with the following fields, where < > indicates an optional field: 
      mass \t protein-id \t peptide-start \t peptide-length <\t peptide-sequence> \n
Options:
  • [--output-sequence] – Output the peptide sequence as well as the protein id and start and stop.
  • [--min-mass >float<] – The minimum neutral mass of the peptides to output Default=200
  • [--max-mass >float<] – The maximum neutral mass of the peptides to output. Default=2400
  • [--min-length <int>] – The minimum length of the peptides to output. Default=6
  • [--max-length <int>] – The maximum length of the peptides to output. Default=50
  • [--cleavages <tryptic | all> ] – Type of cleavages to allow. The tryptic option outputs peptides that result from a complete in silico digestion of proteins after residues K or R except when followed by P, wherease all outputs all possible peptides. Default=tryptic.
  • [--missed-cleavages] – Allow missed cleavage sites with in a peptide. (default = 0)
  • [--sort <p;mass | length | none>] – Specify the order in which peptides are printed to standard output. By default, the output is not sorted.
  • [--isotopic-mass <average | mono>] – Specify the type of isotopic masses to use when calculating the peptide mass. By default, average mass is used.
  • [--verbosity<int>] – Specify the verbosity of the current processes from 0-100. By default, verbosity is set to 100.
  • [--use-index <T|F>] – Specify whether a pre-computed on-disk index should be used for retrieving the peptides from the fasta file (rather than the fasta file itself).
  • [--parameter-file ] The crux parameter file to parse parameter from. (default = crux.params)
  • [--output-trypticity] Output the peptide trypticiy in output. (default = 0)
Warning messages:
  • Fatal: The file "protein input filename" does not exist (or is not readable, or is empty).
  • Fatal: The index for "protein input filename" does not exist (or is not readable, or is empty).
Info:
  • All floats are supported up to 2 decimal digits
  • When using an index database created with all peptide types, only the peptide sources with the correct peptide type are reported. For instance, if user is retrieving tryptic peptides form a database that contains all peptide types, only the tryptic peptide parent sources will be returned.
Bugs:
  • generate-peptides does not have bugs, it has features.
