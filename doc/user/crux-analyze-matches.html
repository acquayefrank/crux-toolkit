crux-analyze-matches

Description:

Given as input a sequence database and PSMs generate additional run-specific features and analyze with user-specified algorithm (e.g. Percolator).

Usage:

crux-analyze-matches  [options]  <match output folder>  <fasta file>

Input:

  • <match output folder> – The name of folder in which all the psm result files are located.
  • <protein input filename> – The name of the file (in fasta format) from which to retrieve proteins and peptides.
Output:

    Returns a reranked peptide list with q-values in sqt file format.
Options:
  • [--algorithm <percolator | retention-czar | all> ] – The analysis algorithm to use. Default = percolator
  • [--parameter-file <filename>] – The crux parameter file to parse parameter from. Default = Default to crux_parameter.
  • [--feature-file <string>] – Optional file in which to write the features (default = none)
  • [--verbosity <int>] – Specify the verbosity of the current processes from 0-100. By default, verbosity is set to 100.
Warning messages:
  • Fatal: The file "folder path name" does not exist (or is not readable, or is empty).
  • Fatal: The file "protein input filename" does not exist (or is not readable, or is empty).
Bugs:
  • match-analysis does not have bugs, it has features.
