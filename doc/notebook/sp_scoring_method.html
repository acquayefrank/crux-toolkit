Proposed Sp scoring method

1.  Process observed spectrum

    1. Square root the intensity of all peaks

    2. Remove peaks with m/z that are greater than experimental_mass + 50

    3. All peak's location(m/z) are rounded to the nearest integer by
      new_m/z = int( old_m/z / bin_width + 0.5 ).
      If there are multiple peaks with the same rounded m/z, choose only the max intensity peak and remove the smaller peaks.

    4. Remove peaks within +/- 15 m/z of the precursor ion

    5. Keep the top 200 abundant peaks, remove others

    6. Normalize the remaining peaks' intensity to 100, so that the max peak intensity equals 100.

2.  Predict fragment ions, use B and Y ions, limit the maximum fragment-ion-charge to precursor-ion-charge -1, if precursor-ion-charge == +1, set max to +1 .

3.  Calculate the Normalized_intensity_sum.

  • For each predicted fragment ion, if there is a matching m/z peak in the processed observed spectrum, add the matching peak's intensity.

  • The total summation off all the matching peak's intensities, equal the Normalized_intensity_sum.

4. Calculate Sp,

  Sp = Normalized_intensity_sum * ni * (1 + m*BETA) / nt     

        ni  = the number of predicted fragment ions that have a matching peak

        nt = the total number of predicted fragmented ions

        m = continuity count of an ion series, ex, if B2-B3-B4 Y3-Y4 match, than m=3

        BETA = 0.075



Comments

  • When calculating the Normalized_intensity_sum, adding the intensity of peaks that have a m/z within +/- 1u of the fragmented ion would be something to try.

  • The order when processing the observed spectrum could have an effect on the Sp score. Especially, when removing peaks outside the top 200, if this occurs before binning the peaks, the top 200 could differ.
    Also, normalizing peak intensities to 100 before removing peaks around the precursor ion could lead to a different score.

  • Still have to resolve the correlation between Sp and unmatched observed peaks.
