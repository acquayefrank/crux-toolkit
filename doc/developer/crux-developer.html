Crux developers' documentation

The Crux developers' directory contains the following:

  • A description of the typical work cycle: change-test-review-commit. This is a good place to start for developers new to crux.
  • The Crux coding standards.
  • The code review process.
  • Information on how CMake is used to build Crux.
  • The Crux API.
  • The Doxygen code documentation Run 'make doc' to produce it.
  • Notes on how new versions are released.
  • A tutorial for working with branches in SVN.
  • Notes on regression, smoke and unit testing.
  • We keep a To Do list on Bugzilla.
  • Some obsolete instructions on how to incorporate C++ into C code.
