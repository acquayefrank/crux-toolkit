Managing the Crux GitHub Project

We use GitHub to host the Crux repository and the Crux website repository. Note that the Crux website repository is automatically updated with content from the doc/user directory of the Crux repository as part of the Crux GitHub build action. You generally shouldn't need to commit anything directly into the Crux website repository.

We used to use TeamCity as our build system. This has been replaced by a GitHub Action that automatically runs whenever a commit is made to the master branch of the Crux repository. Any other branch of the Crux repository that contains .github/workflows/main.yml can also run the build action manually using the GitHub website. One the GitHub website:

  1. Go to the Crux repository.
  2. Choose the branch you wish to work with.
  3. Go to "Actions" for the repository.
  4. In the left sidebar click on the "build-crux-distributions-workflow".
  5. Click on the "Run workflow" button.
Note: the Crux website and release distributions will not be updated when the build action is run on non-master branches.
