search-spectrum

Description:

Given as input an ms2 file, a spectrum scan number within that file, a sequence database, and an optional parameter file, search the spectrum against the peptides in the sequence database, and return high scoring peptides.

Usage:

search-spectrum  <ms2 input filename>  <ms2 scan number>  <protein input filename>  [parameter filename]

Input:

  • <ms2 > – The name of the file (in MS2 format) from which to parse the spectrum.
  • <scan number> – The scan number for the MS-MS spectrum to extract from the MS2 file. This is an integer in the range [1, 100000], and uniquely identifies a particular MS-MS spectrum within a .ms2 file.
  • <protein input filename> – The name of the file (in fasta format) from which to retrieve proteins and peptides.
Output:

    The program prints to standard output a series of header lines, describing how the code were generated, and according to what criteria, as follows: 
      # SPECTRUM FILE: <spectrum input file>
  # SCAN NUMBER: <ms2 scan number>
  # PROTEIN DATABASE: <protein input filename>
Options:
  • [--output-sequence] – Output the peptide sequence as well as the protein id and start and stop.
  • [--min-mass float] – The minimum neutral mass of the peptides to output Default=200
  • [--max-mass float] – The maximum neutral mass of the peptides to output. Default=2400
  • [--min-length int] – The minimum length of the peptides to output. Default=6
  • [--max-length int] – The maximum length of the peptides to output. Default=50
  • [--cleavages tryptic|all ] – Type of cleavages to allow. The tryptic option outputs peptides that result from a complete in silico digestion of proteins after residues K or R except when followed by P, wherease all outputs all possible peptides. Default=tryptic.
  • [--sort mass|length|none] – Specify the order in which peptides are printed to standard output. By default, the output is not sorted.
  • [--isotopic-mass|average|mono] – Specify the type of isotopic masses to use when calculating the peptide mass. By default, average mass is used.
  • [--verbosity int] – Specify the verbosity of the current processes from 0-100. By default, verbosity is set to 100.
  • [--redundancy redundant|unique] – Specify whether peptides that come from different proteins yet with identical sequences should appear on separate lines (redundant) or on the same line (unique).
  • [--use-index T|F] – Specify whether a pre-computed on-disk index should be used for retrieving the peptides from the fasta file (rather than the fasta file itself).
Warning messages:
  • Fatal: The file "protein input filename" does not exist (or is not readable, or is empty).
  • Fatal: The index for "protein input filename" does not exist (or is not readable, or is empty).
Bugs:
  • search-spectrum does not have bugs, it has features.
