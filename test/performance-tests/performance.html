Crux performance test

This test checks how well the Crux search tools work on a sample data set. The data set consists of 7243 spectra, of which 160 are 1+, 7083 are 2+ and 7055 are 3+. Thus, there are a total of 14298 searches to be performed. The database is comprised of 23,932 proteins and 10.5 million amino acids, including C. elegans proteins plus some common contaminants. The test runs three different searches (comet, and tide-search with and without exact p-values) using a 3.0 Da precursor mass window, reversed peptide decoys and a fully tryptic digest. The searches are followed up by analysis via assign-confidence, percolator, q-ranker and barista. The performance of each method is summarized in a plot showing the number of spectra identified as a function of q-value threshold.

You can execute the test by running the script run-performance-test.sh in this directory or the command make performance-test in the parent directory. Assuming that you have gnuplot installed, then the resulting picture should show up in the right-hand panel below:

Here are some things to look for in the output:

  • All the series should start in the lower left corner and be non-decreasing as the q-value threshold is relaxed.
  • Because there are only 7243 distinct spectra, no method should ever find more than this number of PSMs. In practice, the best method is likely to give <3000 PSMs at a q-value threshold of 0.1.
  • No method should leave the y-axis before about 1000 PSMs have been observed.
